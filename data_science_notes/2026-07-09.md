Datos estructurados vs no estructurados.

**Estructurados**: Altamente organizados, con formato rígido (ej. hojas de cálculo, bases de datos SQL).

**Semiestructurados**: No siguen un esquema rígido pero contienen etiquetas o marcadores para separar elementos (ej. JSON, XML, correos electrónicos).

**No estructurados**: Sin formato predefinido ni esquema interno (ej. textos libres, imágenes, videos, publicaciones en redes sociales), requiriendo herramientas avanzadas de IA o NLP para su análisis.

Tipos de datos segun su naturaleza

Comparación de formatos
Como se almacenan los datos:

csv: muy facil de usar, se usa en analisis 

excel: facil uso, formulas, imagenes, tablas dinamicas
se usa en reportes

JSON: JavaScript Object Notation, se usa en apis

EXML: Extensible Markup Language, se usa para integración de herramientas.

SQL: consultas, se usa mucho en empresas


¿Es estructurado, semiestructurado o no estructurado?

Tienen una estructura bien definida y siguen un orden persistente.
Se puede acceder a estos datos y analizarlos en una secuencia lógica. Pueden asignarse rápidamente a campos ya creados y con claves relacionales.
